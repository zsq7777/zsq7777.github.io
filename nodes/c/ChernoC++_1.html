    # 欢迎来到C++

    # C++主要用在那些地方

    C++在计算机系统中的应用有哪些，如游戏，硬件控制，cpu密集型任务中都会使用到他。 为什么？

    # 工作原理

    C++代码->送去编译器->编译器编译目标平台（如：arm64_v8a x86_64）机器码->cpu根据指令运行

    java代码->编译class文件（字节码）->送去java虚拟机运行->java虚拟机运行时翻译为对应平台机器码->cpu根据指令运行

    优势在于c++无需运行时的翻译工作。 但是java语言可以做运行时优化。 这意味着，如果写垃圾代码，他的运行会比java更慢。

    现阶段几乎所有的语言都依赖于C，因为主流操作系统的标准库函数都由有C和汇编完成。

    解释型的语言难点在于解释器，但如果解释器不依赖操作系统，那么语言本身也可以做到不依赖。

    JNode只用了两种语言java和汇编

    市面上通用的语言都实现了语言交互接口(Foreign Function Interface)，简称FFI。 也可以认为他是语言中的通用协议。 C 不再是一种编程语言open in new window原文open in new window

    # 总结

    • 由于C++编译器可编译出系统所应用的平台的可执行二进制文件，无需Java等语言的翻译过程，所以执行效率最高。而且，由于主流操作系统主要由C和汇编编写，那么在这些语言的运行过程中，几乎无法避免使用FFI（语言交互接口）。

    • ABI（应用二进制接口）提供了C/C++库的跨平台被调用的能力。
