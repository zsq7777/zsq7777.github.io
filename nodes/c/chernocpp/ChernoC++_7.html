                                                                                                          # 07. C++链接器是如何工作的

                                                                                                          # Linker的主要作用

                                                                                                          当编译器将文件编译好后，linker找到每个符号和函数在哪里，并将它们链接起来。 即使没有用到外部文件的函数，依然需要linker，程序仍然需要入口，找到main函数，所以仍然需要将main函数链接起来。

                                                                                                          • 编译有两个阶段 编译,linker

                                                                                                          当没有Main函数时，你会发现，编译正确但Linker错误。

                                                                                                          入口点不一是时Main函数，只要是一个入口点就行了

                                                                                                          # static 关键字

                                                                                                          • 第一步： 一个声明了方法但未定义的函数
                                                                                                          • 第二步： 其他方法调用该方法会出现linker错误，虽然该方法未在main函数中使用。 为什么会出现这种错误？ 因为该方法也有可能在其他文件中被使用
                                                                                                          • 第三步：我们使用static关键字，只在当前文件调用，而如果程序中未使用到该函数，那么在linker不生效。

                                                                                                          # 调用函数时，我们需要准确一致的才可以被Linker发现

                                                                                                          返回值 函数名 参数  都必须与调用一致
void Log(char* message)
                                                                                                          1
                                                                                                          2

                                                                                                          # 两个相同的函数

                                                                                                          在一个文件中，编译时就会出现错误。 但如果我们将其中一个函数放入Main文件中编译，就会出现linker错误，linker不知道应该链接到Log.cpp还是Mian.cpp，所以报错。

                                                                                                          # 在头文件中定义函数，不同的cpp文件中使用，也会出现linker错误

                                                                                                          原因：当我们包含头文件时，实际上是将头文件的代码拷贝到该处。

                                                                                                          # 如何解决

                                                                                                          1. static 将函数标记为静态的，linker链接时只意味着这是内部函数。
                                                                                                          2. inline 获取实际的函数体并将函数调用替换为函数体。
                                                                                                          3. 不将函数定义至头文件，而只做函数声明，将函数定义至cpp文件中，在调用时include头文件调用函数。

                                                                                                          # 动态链接 静态链接等

                                                                                                          # 总结

                                                                                                          linker的关键在于cpp文件及其函数之间的调用，为了达成这样的功能。需要一种寻找机制，如函数唯一性，多cpp文件引入头文件。 static关键字 在标记函数时，表示该数据成员那就是类内的静态数据成员。
