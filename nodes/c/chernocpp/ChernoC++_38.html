                                                                                                                # 38. 创建并初始化C++对象

                                                                                                                # 初始化C++对象的区别

                                                                                                                内存从哪来，我们从哪创建对象

                                                                                                                # 类对内存的影响

                                                                                                                一个类什么也没有也会占用1个字节的内存

                                                                                                                # 内存主要分为两部分，栈和堆 ，还有其他部分内存，如源代码区域

                                                                                                                # 栈对象

                                                                                                                一个自动的生存期，生存期实际上由它声明的地方作用域决定的，变量超出作用域就被释放了。 栈对象的生存期由它所声明的作用域决定。

                                                                                                                # 创建方法

                                                                                                                通常应该使用这种方法创建对象，因为这是C++中最快的方法，也是可以”管控的方法“，去初始化对象。

                                                                                                                Entity entity;
Entity entity1("赵思琦");
Entity entity2=Entity("小赵");
std::cout << entity2.getName() << std::endl;

                                                                                                                # 堆对象

                                                                                                                堆中分配一个对象，之际上已经在堆上创建了一个对象，知道做出决定不再需要它，我需要释放。 堆对象的生存期，除非手动释放，不然会一直存在。

                                                                                                                Entity* entityW = new Entity("赵思琦");
                                                                                                                1

                                                                                                                这就是Java或C++中创建对象的样子。

                                                                                                                # 与Java/C#的区别

                                                                                                                C++额外可以在栈上分配。 C#中的struct，基于值类型，他们实际上时在栈上分配的，即使使用new关键字。C#所有类都是在栈上分配的 Java所有的东西都在堆上

                                                                                                                # 堆上分配的性能问题

                                                                                                                堆上分配内存要比栈花费更长时间，而且如果在堆上分配的话，必须手动释放被分配的内存

                                                                                                                # 删除该对象所占内存

                                                                                                                delete entityW;
                                                                                                                1

                                                                                                                # 使用不同方法创建对象的情况

                                                                                                                类比较大，需要显式控制对象生存期，就用堆上创建。否则 就用栈上创建

                                                                                                                # 总结

                                                                                                                栈上的对象的生存期与变量所声明的作用域有关。作用域结束， 栈帧销毁。 堆上创建的对象需要自己手动释放内存。 堆对象需要new关键字，栈不用。 栈对象的创建更快，且易于管理，但是可用内存小。 堆对象生存期更长，但要注意控制，可用内存大。
