                            # 01. 序列化方案

                            序列化在程序中的应用无处不在，它是计算机软件通信重要的一部分，应用范围包括网络传输，文件及数据库持久化，进程间通信等，RIM(远程方法调用)。

                            序列化：将对象转化为二进制流的过程。

                            # 列举常见的序列化和反序列化方案（ObjectXXStream、XML、JSON）

                            # ObjectXXXStream

                            # Serializable

                            这是一个标记接口，ObjectOutputSteam/ObjectInputStream完成对象的序列化/反序列化。通过反射实现，过程慢，序列化时产生了大量临时变量，引起频繁GC。但易于持久化，但是由于他的转换规则是根据.class文件，所以只能在java中使用。

                            # Parcelable

                            Android原生序列化，使用native方法，对指针的操作。 内存中完成序列化与反序列化。用于Intent数据传递，进程间通信IPC。

                            # XML、JSON

                            • 他们都具有人眼可读的特性。
                            • 都必会经历 对象-> String->byte[]的过程，因为数据类型的问题，数据放大。
                            • 包含Object类型，业务本身也不明确，处理起来会很麻烦。

                            # 概念

                            IDL ：通讯各方对通讯内容的约定

                            # 数据放大

                            JSON根据转换原理 ，对象原本long类型的数据在内存中占用8个字节，转化为字符串，数据放大到了30字节,3.78倍。

                            # 数据放大的问题是什么

                            1. 花费时间去拼接字符串，拷贝内存，占用更多的java内存，产生更多内存碎片。
                            2. 对象到string再到byte[]必然要多一次全量内存拷贝
                            3. 传输数据放大，更占流量
                            4. package变多，在网络延迟高，丢包可能性更大。
                            5. 接收方花更多的时间处理。
                            6. string与其他类型的转换，额外的内存和计算开销。
